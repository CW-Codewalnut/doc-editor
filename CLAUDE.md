# Project Guidelines

## Commit Convention
- Use conventional commits: `feat:`, `fix:`, `refactor:`, `chore:`, `docs:`, `test:`, `style:`

## Code Style
- No single-character variables — use descriptive names
- DRY principle — don't repeat yourself
- KISS principle — keep it stupid simple
- Single Responsibility Principle — one purpose per function/component/module
- Functional React components with hooks only

## Tooling
- Use `bun` instead of `npm` for all package management and script execution (`bun install`, `bun run`, `bunx`, etc.)
- Use GitHub CLI (`gh`) for all GitHub operations (PRs, issues, repos, etc.)
